# نظام إدارة عقود الوساطة والتراخيص الإعلانية

مشروع **Laravel 12 كامل وجاهز للتشغيل** لإدارة عقود الوساطة العقارية والتراخيص الإعلانية، مع إشعارات تلقائية (واجهة + بريد)، لوحة معلومات احترافية، صلاحيات للأدوار، وواجهة عربية كاملة بدعم RTL.

> هذه الحزمة تحتوي على **هيكل Laravel الكامل** (`artisan`, `composer.json`, `bootstrap/`, `public/`, `config/` …) بالإضافة إلى ملفات النظام. كل ما يلزم هو تثبيت اعتماديات Composer ثم الترحيل.

---

## 1) المتطلبات
- PHP 8.2 أو أحدث
- Composer
- إضافات PHP المعتادة (`pdo_sqlite` مفعّلة افتراضياً، أو `pdo_mysql` عند استخدام MySQL)

---

## 2) التشغيل السريع (3 خطوات)

```bash
# 1) تثبيت الاعتماديات (Laravel + Livewire + Spatie Permission)
composer install

# 2) الترحيل وتعبئة البيانات التجريبية
php artisan migrate --seed

# 3) تشغيل الخادم
php artisan serve
```

ثم افتح: `http://localhost:8000`

- ملف `.env` **جاهز ومضبوط مسبقاً** على قاعدة بيانات **SQLite** (لا يحتاج أي إعداد)، ومفتاح `APP_KEY` مولّد بالفعل.
- ملف `database/database.sqlite` مُنشأ ومضمّن.

> إن أردت إعادة توليد المفتاح: `php artisan key:generate`

## تحسينات الأداء والمسؤول عن العقد

- **المسؤول عن العقد حقل حرّ:** لم يعد مرتبطاً بمستخدم في النظام — أصبح (اسم المسؤول + جوال) يُدخلان يدوياً، وقابلاً للفلترة في جدول العقود والتقارير. أما نطاق «بيانات الموظف الخاصة» في لوحة المعلومات فأصبح مبنياً على ما أنشأه الموظف والتراخيص التي أصدرها.
- **تسريع تحميل الصفحات (الأهم):** استُبدل Tailwind عبر CDN (الذي كان يُصرّف الأنماط داخل المتصفح في كل تحميل) بملف CSS **مُصرّف ومصغّر مسبقاً** (`public/css/app.css` ~41KB). هذا يزيل عبئاً كبيراً عن كل صفحة. لإعادة البناء بعد تعديل الواجهات: `npm install && npm run build:css`.

## تحديثات الشركات الخارجية والتقارير والأداء

- **الشركات الخارجية:** كيان مُدار (اسم الشركة، الشخص المسؤول، الجوال) للعقود الفرعية — بشاشة إدارة مستقلة (صلاحية «إدارة الشركات الخارجية»)، ويُعاد استخدامه تلقائياً بمطابقة الاسم، ويظهر في صفحة العقد الفرعي وكفلتر في تقرير العقود.
- **تقرير التراخيص لكل مستخدم:** مبدّل في واجهة التقارير بين «تقرير العقود» و«تقرير التراخيص»؛ الأخير يتيح تقريراً لتراخيص أي موظف/مستخدم أنشأ ترخيصاً (حتى لو لم يكن مسؤول العقد) مع فلاتر التاريخ وحالة الترخيص وحالة النشر.
- **تحسين الأداء:** تخزين مؤقت لأسماء المنصات النشطة لمنع استعلامات N+1 عند حساب حالة النشر، وeager loading في كل القوائم، وفهارس على أعمدة الفلترة (platform_count, status, issue_date, الشركة الخارجية…).

## تحديثات إدارية وتنظيمية

- **إدارة المستخدمين:** يمكن للأدمن تغيير كلمة مرور أي مستخدم أو حذفه (مع منع حذف الحساب الشخصي).
- **فلاتر باسم الموظف:** أُضيف فلتر «اسم الموظف» في واجهة التراخيص، وهو متوفر في التقارير.
- **نوع الصفقة:** لكل عقد وساطة نوع صفقة **إيجار / بيع** (يظهر في النموذج والتقارير كفلتر).
- **العقود الفرعية:** إمكانية اشتقاق **عقد فرعي** من عقد قائم لشركة أخرى — ينسخ بياناته بالكامل ويأخذ رقم عقد جديد، **بصلاحية مستقلة** (`إنشاء عقد فرعي`) تُمنح لأي موظف، ولا يتطلّب ترخيصاً ولا يُرسل إشعارات (غرضه تنظيمي).
- **صلاحية إدارة العقود:** أي موظف يُمنح صلاحية «إدارة العقود» يرى الآن **كل العقود بكل حالاتها** (كان محصوراً بدور المدير).

## تحديثات الإصدار الأحدث

- **الهوية البصرية:** اسم المنصة **«سهل»** بشعار وألوان جديدة (كحلي + سماوي).
- **أرقام يدوية:** رقم العقد ورقم الترخيص يُدخلان يدوياً من الموظف (يُجلبان من المنصة الأخرى) مع التحقق من عدم التكرار.
- **روابط الإعلان إجبارية:** عند تحديد منصات النشر يجب إدخال رابط الإعلان لكل منصة.
- **متابعة النشر:** تمييز التراخيص (لم يُنشر / نشر جزئي / منشور بالكامل) وإظهار العقود بلا ترخيص.
- **تنبيهات الفجوات:** إشعار الموظف بالعقود بلا ترخيص أو التراخيص غير المنشورة بالكامل.
- **حالات جديدة:** حالة عقد «منتهي»، وحالة ترخيص «تم الإنشاء ولم يتم النشر».
- **فلاتر إضافية:** فترة تاريخ في واجهة التراخيص، وفلتر حالة الترخيص/النشر في التقارير.

## الجديد في هذا الإصدار

- **عقد واحد ← عدة تراخيص:** المشرف ينشئ العقد (رقمه يبدأ بـ 62) فيُشعَر كل الموظفين، ويُنشئ كل موظف ترخيصاً واحداً فقط لنفس العقد (رقمه يبدأ بـ 72). العقد منفصل عن الترخيص.
- **حالات العقد:** في انتظار الموافقة / تمت الموافقة / انتهت المدة دون موافقة / ملغي.
- **تراخيص لكل موظف:** إضافة وتعديل وحذف لكل ترخيص (للموظف ترخيصه، وللمدير إدارة الجميع).
- **تبويبة الأدوار والصلاحيات:** إنشاء أدوار وتخصيص صلاحياتها ديناميكياً (Spatie).
- **ملف شخصي لكل مستخدم:** عرض وتعديل البيانات وكلمة المرور.
- **واجهة تقارير:** فلاتر بالتاريخ والمسؤول والمندوب والحي والنوع والحالة.
- **حقول العقد الجديدة:** المندوب (مع إدارة كاملة للمناديب)، المسؤول عن العقد، نوع العقد (حصري/وساطة/تسويق)، الحي — وكلها فلاتر في التقارير.
- **لوحة معلومات مُخصّصة:** كل موظف يرى بياناته الخاصة فقط.

> عند إعادة التشغيل بعد التحديث استخدم `php artisan migrate:fresh --seed` لأن بنية الجداول تغيّرت.

## 4) التحويل إلى MySQL (اختياري)

في ملف `.env` عطّل سطر sqlite وفعّل أسطر mysql:

```dotenv

DB_PORT=3306
DB_DATABASE=aqar
DB_USERNAME=root
DB_PASSWORD=
```
ثم: `php artisan migrate:fresh --seed`

## 5) المهام التلقائية (تحديث الحالات + تنبيهات الانتهاء)

الأمر `contracts:check`:
- يحوّل العقود المعلّقة إلى منتهية بعد تجاوز المهلة (من الإعدادات).
- يحوّل العقود التي تجاوزت `end_date` إلى منتهية.
- ينشئ إشعار `expiring_license` (واجهة + بريد) للتراخيص المقتربة من الانتهاء.

تشغيل يدوي:
```bash
php artisan contracts:check
```
مجدول تلقائياً يومياً 08:00 (في `routes/console.php`). لتفعيل الجدولة على الخادم أضف إلى cron:
```bash
* * * * * cd /path-to-project && php artisan schedule:run >> /dev/null 2>&1
```

## 6) البريد والإشعارات

- افتراضياً `MAIL_MAILER=log` و`QUEUE_CONNECTION=sync` → رسائل البريد **تُسجَّل فوراً** في `storage/logs/laravel.log` دون الحاجة لخادم بريد أو مشغّل طابور (مثالي للتجربة).
- للإرسال الفعلي: غيّر `MAIL_MAILER=smtp` واضبط `MAIL_HOST/PORT/USERNAME/PASSWORD` في `.env`.
